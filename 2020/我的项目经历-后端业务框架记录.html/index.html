装载路由器

  • 使用的就是gin的路由。

  • 用了四个中间件。

    • Session

      • session必须依赖于cookie才能使用，生成一个sessionId放到cookie里面传递给client
      • 用户验证这种场合一般会用 Session。因此，维持一个会话的核心就是客户端的唯一标识，即Session ID。

  • CurrentUser 用于获取session的user。用上下文c里包含的gin-session生成一个session数据结构，获取本数据结构的user_id。然后查看这个user_id是否能在数据库查到。如果可以，说明已经顺利登录。

    • ```GO
      // CurrentUser 获取登录用户
      func CurrentUser() gin.HandlerFunc {

      return func(c *gin.Context) {
    session := sessions.Default(c)
    uid := session.Get("user_id")
    if uid != nil {
        user, err := model.GetUser(uid)
        if err == nil {
            c.Set("user", &user)
        }
    }
    c.Next()
}

      }

- 等我们需要使用**AuthRequired** 中间件保护的接口的时候，浏览器会携带该cookies进行api访问。



- 当我们logout的时候，也会清理掉这个session

  • 所以总结来说，如何进行鉴权？

    • 后端对每个请求，都会通过它的cookies里面包含的session获取user_id
    • 通过user_id查询数据库是否存在该用户，如果查询成功，就将当前gin.Context的user设置为查询到的user。否则就为空。
    • 当访问需要登录的接口的时候，就会查询gin.Context里面的user是否存在，如果存在就可以访问；否则就不能访问。
    • 用户的登录会产生一个session信息在reponse的cookies，登出会消除掉这个session信息。

  • Cors 是解决跨域问题的

开发接口的流程是怎么样的？

  1. 在router中绑定url以及响应函数
  1. 在api文件夹新建user.go，创建响应函数

每个响应函数都有不同的处理。有的需要传参，有的不需要。

  • 如果传参，就需要用到 gin里面的ShouldBind 进行参数解析。
  • 如果不传参，就不需要用（也别用，否则会报错）。
  1. 可以看到在第二步的时候，有进行服务的初始化。所以为了完成第二步，事实上也需要code所需要的服务。

以上传文章为例，我们在service文件夹创建article_post_service.go

  • 在ShouldBind 的时候，会对sevice进行数据绑定。为了正确绑定，我们要定义Service的结构
  • 结构说明
    • Title必须大写，才可以被外面访问。
    • tag内容有说明想要绑定的数据是以什么格式从前端传来后端的，binding对字段进行校验。
      • form:title。其中form表示表单，title表示传来的字段名字。
      • binding后就是对输入属性的具体要求。required表示必选，也说明了最小值和最大值。

  • 然后需要定义针对结构体的方法函数 可能用于进一步验证，也可能用于真正的业务处理。下面两个函数，分别说明。
